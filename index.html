maven-growl

The Problem

If you spend time in Maven, then you're probably using Java or at least significant parts of the Java platform, and you're probably familiar with builds that take a while to run. If your build takes longer than a few seconds to run, then you probably switch your attention to something else while your build runs. You could check your email, catch up on your RSS feeds, play a casual game, or do something else. This has its merits, but it tends to disrupt your flow. Worse still, if your build fails quickly, you might not realize it because you won't even return to check on it 'til you imagine that it might have finished. Accordingly, it's helpful to have something let you know that the build is complete, or has failed, as soon as possible, so you can get back to developing and building software.

If you're developing on the OS X platform, you're probably already familiar with Growl. Growl is configurable, powerful, and pervasive. Many applications you're already using integrate with Growl to send notifications. It would be nice if Maven could integrate with Growl as well.

Solutions

If you're already using Maven 3.x and the Maven shell, you're in luck, @jvanzyl says that the shell already supports growl notifications using grrrowl. For the rest of us, some kind of bridge is needed to pipe Maven status into Growl.

A quick Google search will show you that there are already some approaches to bridge Maven to Growl. You could use awk to parse the Maven build output, and then call growlnotify" to send a notification.

Using awk works, but it seems like the exit code from Maven (0==success, >0==failure) is simpler to parse, so I've written up a short bash script that executes Maven, checks the return code, then calls growlnotify with success/failure messages and a simple red/green image.

Using maven-growl

This isn't a finished product, it's a very small set of files that you can use to assemble your own solution. Start by taking a look at the simple bash script I'm using to invoke Maven and call growlnotify. You could put this somewhere in your path, or write a script of your own. If you want to use the images, put those somewhere and modify the script to find them. You'll also need to install growlnotify, an optional install that comes in the extras with Growl. That's basically it, you're done.
